Phase 1 (30/4) — Chuyển tiền → Thanh toán

🔍 Nate — Analysis

UX RESEARCHER

User Flow

Home → Tab TT → category pill → chọn DV (list row) → chọn NCC → nhập mã KH → auto-fetch bill → confirm → biometric → result. 5-7 taps. Returning user với saved biller: 3 taps (tab → saved → confirm).

Mental Model Validation

"Tiền di chuyển" (Home: Nạp/Rút/Chuyển/Nhận) vs "Trả cho dịch vụ" (Tab TT). Tách rõ, không overlap. Chuyển tiền qua NH = "gửi cho ai đó" (mode 1) → đúng ở Home.

User kêu chuyển tiền khó tìm → đưa lên Home quick action = solved. Thêm recent recipients (2-3 avatars) dưới quick actions cho returning users.

Key Decisions

  • HST = LIST (không phải grid) — brand recognition cần logo lớn + description per partner. Grid 4 items quá nhỏ, không nổi bật. DV categories (Điện/Nước) = grid vì generic, chỉ cần icon + label.
  • Saved billers = TRƯỚC grid (đổi thứ tự: Search → Urgent banner → Đã lưu → Grid → Nạp nhanh → Gần đây). Lý do: 80% user returning muốn trả bill đã có, không phải đăng ký mới.
  • Category grid: Điện / Nước / Internet / Nạp ĐT / Thẻ cào / Gói data / Khoản vay / Khác. Đổi "Tài chính" → "Khoản vay" (rõ hơn). Giữ tách 3 entries di động (mỗi flow input khác nhau — gộp = thêm 1 tap).
  • Label ambiguity fix: Home quick action = "Nạp ví" (không phải "Nạp"). Tab TT = "Nạp điện thoại" (không phải "Nạp tiền"). Tránh user confuse giữa nạp ví vs nạp ĐT.
  • Search scope: Categories + NCC cross-category + HST partners + Saved billers. KHÔNG search mã KH (vô nghĩa ngoài context NCC). Saved billers lên trước trong kết quả.
  • Quick actions PHẢI above the fold — fixed row dưới wallet card, không scroll mất. Nếu Home content push xuống → "Chuyển" biến mất → user mất.

👹 Đức — Review Phase 1

SENIOR REVIEWER — 7/10 PASS WITH FIXES

Issue 1: Muscle memory conflict. Tab Thanh toán nằm đúng vị trí tab Chuyển tiền cũ (tab 2). User tap theo habit → vào Thanh toán thay vì Chuyển tiền. Coach mark 1 lần không đủ — user sẽ quên sau 2 ngày.

Cách sửa: Tab Thanh toán lần đầu mở → inline banner top: "Bạn muốn chuyển tiền? Tap 'Chuyển' trên Trang chủ ↗" + deep link. Banner tự ẩn sau 7 ngày hoặc sau khi user tap "Đã hiểu".

Issue 2: Saved billers empty cho first-time user. Section "Đã lưu" ở top = perfect cho returning. Nhưng first-time user mở Tab TT → thấy section trống → bad first impression.

Cách sửa: First-time: ẩn section "Đã lưu", grid categories lên top. Returning (≥1 saved): "Đã lưu" lên top, grid xuống dưới. Layout dynamic theo state.

Issue 3: "Di động" category chưa cover mua thẻ game. Grid có "Nạp ĐT / Thẻ cào / Gói data" — nhưng thẻ game (Garena, Google Play) nằm đâu?

Cách sửa: Slot "Khác" hoặc gộp "Thẻ cào" → "Mua thẻ" (bao gồm thẻ ĐT + thẻ game).

Phase 2 (Q3) — Giao dịch → Tài chính

🔍 Nate — Analysis

UX RESEARCHER

Financial Hub Structure

Hero: Tổng tài sản = assets thật (Ví + SL + BĐS + CCQ). BH hiện riêng "Đang bảo vệ: X sản phẩm" — BH không phải tài sản. BNPL hiện "Dư nợ: X" — liability, không cộng vào tổng.

Ordering: Dynamic by AUM (product user có nhiều tiền nhất → đầu tiên). Default user mới: SL → BH → BĐS → CCQ → BNPL → Vay VinFast. Không cần sub-nav — scroll vertical đủ.

Section grouping: "Tích luỹ" (SL, CCQ) / "Đầu tư" (BĐS) / "Bảo vệ" (BH) / "Tín dụng" (BNPL, Vay)

GD Merge vào Home

Wallet Card + "GD gần đây" 3 items luôn visible trên Home. Tap wallet / "Xem tất cả" → fullscreen GD với search + filter pills (7 categories) + grouped by date. Tap count = bằng tab GD cũ.

Key Findings

  • Portfolio 2 levels: Glanceable (hero card, không chart) → Tap → Full screen (chart + breakdown). Chart chỉ khi ≥2 products.
  • Cross-sell: KHÔNG trong product detail. Max 2 discovery cards ở bottom tab level. Dismissible.
  • Graduation: Big bang — tất cả user đổi cùng lúc per segment. Không 2 UI song song (QA nightmare).
  • Widget tap → product detail page (push navigation từ Home). Không tab switch.
  • "Tài chính" tên đúng — MoMo/ZaloPay đã train mental model VN. Section titles bên trong phân loại rõ.
  • Habit break mất tab GD = MEDIUM-HIGH. Mitigation: Home GD section visible, wallet affordance, transition BottomSheet 1 lần.

Lofi Wireframe — Phase 2

Tab Tài chính

👹 Đức — Review Phase 2

SENIOR REVIEWER — 6/10 PASS WITH FIXES

Issue 1: Mất tab GD = user VN hoảng. Severity MEDIUM-HIGH nhưng mitigation (1 lần BottomSheet) quá nhẹ. User VN check GD SAU MỖI giao dịch — đây là habit hàng ngày, không phải hàng tuần.

Cách sửa: Home "GD gần đây" section phải LUÔN visible (không bị push xuống bởi widgets). Wallet card phải có số GD hôm nay visible ("3 GD hôm nay"). Transition BottomSheet không đủ — cần inline guide TRONG Home liên tục 14 ngày.

Issue 2: Hero "Tổng tài sản" show số nhỏ cho user mới. User mới chỉ có 14.328đ trong ví, chưa có SL/BĐS. Tổng tài sản = 14.328đ hiện lớn trên hero card → feels lame, turn off.

Cách sửa: User có <1 product active → hero card chuyển sang discovery mode: "Bắt đầu hành trình tài chính" + product cards. Không show tổng tài sản khi số = balance ví.

Issue 3: Widget → Tab handoff chưa chốt. Nate để 3 options (push from Home / tab switch / modal) mà không recommend 1. PO không phải UX designer — đừng đẩy decision cho PO.

Cách sửa: Chốt: tap widget → push navigation (present product detail page from Home context). Back = quay Home. Đơn giản nhất, user hiểu nhất.

Phase 3 (Q4) — Tài khoản → Ưu đãi

🔍 Nate — Analysis

UX RESEARCHER

Tab Structure

Hero: V-Point balance card (điểm + "Đổi quà") — anchor. Dưới: Hot deal (1 featured) → Category pills → Deal list → Referral card.

Ordering: V-Point (sở hữu) → Featured deal (urgency) → Browse (discovery) → Referral (acquisition).

V-Point UX

Tích: tự động khi thanh toán, toast "Bạn nhận 25 V-Point ✓". Dùng: toggle "Dùng V-Point" ở checkout. Đổi: tab Ưu đãi → "Đổi quà" → chọn → confirm. Edge: điểm 0 → CTA "Thanh toán để tích điểm".

Avatar Pattern — Risk Assessment

Tài khoản → avatar góc phải (Cash App). Tap count: "Liên kết NH" = 2 taps (avatar → LKNH). Trước = 2 taps (tab TK → LKNH). Bằng nhau. Risk = discoverability, không phải accessibility. Mitigation: badge notification trên avatar, tooltip lần đầu.

Key Findings

  • Ưu đãi vs Thanh toán overlap: Tab Ưu đãi = browse + save. Tab TT = use. Không cross-navigate. User save deal → checkout tự apply.
  • Engagement loop: Earn V-Point → check → gần đủ → dùng VSP thêm → earn → loop. Cần earn rate đủ nhanh (~100 VP/giao dịch).
  • Categories: Tất cả / Vingroup / Ăn uống / Di chuyển / Mua sắm / Du lịch. "Vingroup" riêng = USP.
  • Deals curation: Q4 = manual. 2027 = personalized (HST usage + transaction history + location).
  • Revenue: V-Point breakage ~200M/năm + Partner commission ~250M/tháng. Self-sustaining nếu ≥50K active deal users.

Lofi Wireframe — Phase 3

Tab Ưu đãi

👹 Đức — Review Phase 3

SENIOR REVIEWER — 4/10 REWORK

Issue 1: Avatar thay tab Tài khoản = cultural mismatch. 100% app VN (MoMo, ZaloPay, VNPay, Viettel Money) đều có tab Tài khoản ở bottom nav. Cash App pattern là US — user VN chưa có learned behavior này. Đây không phải "habit break nhẹ" — đây là phá convention toàn thị trường.

Cách sửa: KHÔNG move Tài khoản lên avatar ở Q4. Giữ tab Tài khoản. Nếu muốn thêm Ưu đãi → tìm cách khác (xem verdict bên dưới).

Issue 2: Dead tab risk. Tab Ưu đãi phụ thuộc deals quality + V-Point earn rate — cả 2 do business team quyết, UX không control. Nếu deals lặp, V-Point earn chậm → tab trở thành dead space → user skip → MAU không tăng mà còn gây ấn tượng "app trống".

Cách sửa: Không tạo tab riêng cho content mà team không control. Ưu đãi = section trên Home (widget) hoặc entry point trong Tab Thanh toán ("Dùng ưu đãi khi thanh toán").

Issue 3: Voucher cross-tab flow thiếu auto-suggest. User save deal ở tab Ưu đãi → checkout ở tab TT → deal tự apply. Nhưng ngược lại? User ở tab TT checkout XanhSM → KHÔNG biết có deal XanhSM -50% → miss. Phải auto-suggest deals relevant tại checkout.

👹 Đức — Verdict Toàn Strategy

OVERALL ASSESSMENT
"Tab bar thay đổi 3 lần trong 6 tháng. App tài chính = trust = stability. Mày là user, mày có tin app mà cứ thay đổi liên tục không?"

Đề xuất: Lock tab bar từ Phase 1.

Thay vì đổi tab 3 lần, chọn tab bar CUỐI CÙNG ngay Phase 1 và chỉ thay đổi CONTENT bên trong:

Phase 1 (30/4):  Trang chủ — Thanh toán — QR — Tài chính — Tài khoản
                 Tab Tài chính: chỉ có SL teaser + BH teaser (ít nhưng OK)
                 Tab Tài khoản: giữ nguyên

Phase 2 (Q3):    CÙNG TAB BAR
                 Tab Tài chính: += BĐS + CCQ + BNPL (fill dần)

Phase 3 (Q4):    CÙNG TAB BAR
                 Ưu đãi = section trong Home widgets hoặc entry trong Thanh toán
                 Tab Tài khoản: += V-Point section

Trade-off: Tab Tài chính 30/4 hơi ít content (chỉ BH + BĐS teaser + SL teaser). Nhưng STABLE > FULL. User trust app không đổi layout > user thấy tab đầy content.

5 Edge Cases Cross-Phase Nate Bỏ Sót

  1. Multi-device rollout: User dùng 2 device → 1 cái update Phase 2, 1 cái vẫn Phase 1 → confuse.
  2. Force update skip: User skip update → vẫn ở Phase 1 khi Phase 2 đã launch → backend API mismatch?
  3. Accessibility user lớn tuổi: Vinhomes có nhiều cư dân >50 tuổi. Đổi tab bar = họ gọi con cháu hỏi. Frequency: cao.
  4. Offline mode: Mất mạng giữa tab switch → tab mới load trống → user nghĩ app hỏng.
  5. Deep link version-aware: Link chia sẻ "vsppay://finance/sinhloi" → user ở Phase 1 chưa có tab Tài chính → 404? Redirect đâu?

User Needs

Nhu cầu User nói Entry Point
① Thanh toán
80% traffic		 "Đóng tiền điện" Quick Pay card → pay ngay
② Thêm mới
20% traffic		 "Lần đầu đóng tiền nước" Category grid → full flow → "Lưu?"
③ Quản lý
5% traffic		 "Đổi tên / xóa / xem chi tiết" Tap card → Biller Detail
"Xem tất cả" → Full list
Lịch sử GD: Icon ⏱ trên NavBar → navigate sang Lịch sử GD chung. Không render trong VAS.
Auto-pay / Nhắc nợ / Nạp tự động: Phase 2 (sau 30/4). Phase 1 chỉ cần suggestion sau khi TT thành công.
Header thay đổi theo state: New user = "Điện, nước, internet — thanh toán tại đây" (context). Returning user = "Dịch vụ" (đã có Đã lưu section cho context).

Flow Map

① Returning user (đã lưu): Home → Quick Pay card → [Thanh toán] → Confirm sheet → Auth → Result 3 taps, <15 giây ② First-time (hóa đơn mới): Home → Danh mục → Chọn NCC → Nhập mã → Bill Detail → Confirm → Auth → Result └─ ☑ "Lưu hóa đơn này" ③ Quản lý: Home → Swipe row → Xóa Home → Tap row → V4 (Bill Detail) → pay / xem info "Xem tất cả (N)" → overflow list (nếu >5 items)

Tài chính
V1 — Home (new user)
Section "Đã lưu" có 2 tabs: Hóa đơn | Số điện thoại. Chỉ xuất hiện khi user có saved items. New user chỉ thấy Category Grid.
Tab Hóa đơn: Quick Pay cards (EVN, Nước, FPT...) — tap → Bill Detail → pay hoặc quản lý.
Tab SĐT: Saved phone cards — tap "Nạp tiền" → Topup flow pre-filled.
"Xem tất cả (N)" chỉ hiện khi >5 items → overflow list cùng format. Tap row → V4 pay. Swipe left → xóa.

1. Revolut — Financial OS Model

1.1 Bottom Nav (5 tabs)

TabChức năngInsight
HomeBalance hero (40px), quick actions (4 circles), widgets user-customizable, recent transactionsBalance là hero lớn nhất. Home = personal dashboard, user tự chọn hiện gì.
TransferSend/receive, payments, bill splittingCore use case giữ riêng tab — origin story của Revolut
HubALL products on 1 page, grouped by category (Essentials / Wealth / Lifestyle)Key pattern: Hub = feature catalog. Mọi sản phẩm mới vào Hub trước.
WealthStocks, crypto, commodities, savings vaults, robo-advisorPromoted lên tab riêng vì là revenue driver lớn
ProfileSettings, security, accountStandard

1.2 Revolut Scaling Formula

Progressive Promotion Ladder: 1. Sản phẩm mới → vào Hub (catalog) dưới category phù hợp 2. Sản phẩm có traction → lên Home dạng promotional card / widget 3. Sản phẩm thành core → promote lên Bottom nav tab 4. Sản phẩm thành business → spin ra Standalone app (Revolut X, Revolut Invest) 3-layer architecture: Layer 1: Home (personalized) — user chọn hiện gì, pin/unpin Layer 2: Hub (full catalog) — tất cả products, grouped + searchable Layer 3: Dedicated tabs — chỉ cho high-value verticals (Wealth, Transfer)
Insight cho VSP: Revolut KHÔNG dùng service grid (icon grid MoMo-style). Thay vào đó dùng product cards + widgets. Lý do: khi có >10 products, icon grid trở nên overwhelming. Cards cho phép hierarchy rõ hơn (primary/secondary/coming soon).

3. So sánh 3 Models

🔵 Revolut
Financial OS
  • Products as cards
  • Hub = full catalog
  • Wealth tab riêng
  • User tự customize Home
  • Clean, minimal
  • Scale: 30+ products OK
🟢 ZaloPay
Super App
  • Service grid (icons)
  • Tab Ưu đãi (deals)
  • No financial tab
  • Grid scale bằng thêm icon
  • Familiar cho VN users
  • Products bị chôn sâu
🟡 VSP
Hybrid?
  • Hiện tại = basic wallet
  • Cần scale lên 4 verticals
  • User VN quen icon grid
  • Nhưng ko muốn MoMo clone
  • Financial products = key diff
  • → Revolut approach

4. VSP Decision: Revolut Financial OS

4.1 Tại sao chọn Revolut model, không ZaloPay?

Tiêu chíZaloPay (Grid)Revolut (Cards)VSP chọn
Financial products visibility Chôn trong grid → sub-page. Cần banner để push. Product cards ngay Home + Wealth tab riêng Revolut — VSP cần push Sinh lời + Đầu tư
Scale 4 verticals Thêm icon vào grid → grid 4×5 → overwhelming Hub catalog + promote system. Clean ở mọi scale. Revolut — clean khi add Tín dụng + BH
Differentiation Giống MoMo / VNPay → không có brand identity Khác biệt hoàn toàn. Premium feel. Revolut — VSP cần khác MoMo
VN user familiarity Quen thuộc — MoMo/ZaloPay/VNPay đều dùng grid Mới lạ — cần onboarding Trade-off — chấp nhận curve learning nhỏ
Development cost Grid đơn giản, thêm icon dễ Cards phức tạp hơn nhưng reusable Revolut — card component = 1 lần build, reuse mãi

4.2 VSP Bottom Nav — Final Decision

VSP Bottom Nav (5 tabs) — Revolut-inspired: 🏠 Trang chủ | 💰 Tài chính | 📱 QR (center) | 🕐 Giao dịch | 👤 Tài khoản Mapping to Revolut: Revolut Home → VSP Trang chủ (balance, quick actions, product cards, GD) Revolut Wealth → VSP Tài chính (tổng tài sản, sinh lời, đầu tư, BNPL, BH) Revolut Transfer → VSP QR (combine scan + show + chuyển tiền quick) Revolut Hub → KHÔNG CẦN ở 30/4 (VSP chưa có 30+ products) Revolut Profile → VSP Tài khoản Khi nào cần Hub? Khi VSP có >15 services (dự kiến Q4 2026 hoặc 2027). Lúc đó: Giao dịch → merge vào Home, Hub lên thay tab Giao dịch.

7. Verdict

VSP = Revolut for Vietnam, không phải MoMo clone

  • 30/4: Ship Financial OS shell — Home v2 (product cards) + Tab Tài chính (Sinh lời + BĐS teaser) + New bottom nav. Chỉ show cái đã có.
  • Q3: Fill Tài chính tab — BĐS live, portfolio view, HST cards trên Home.
  • Q4: Full 4 verticals — BNPL + BH vào Tài chính tab. Loyalty integrate Home cards.
  • 2027: Personalization layer — widgets, pin/unpin, Hub (nếu >15 services), smart search.

Core principle: Chỉ show sản phẩm đã có. Không placeholder "Sắp ra mắt". Khi product ready → add card. Cards scale vô hạn mà không break IA.

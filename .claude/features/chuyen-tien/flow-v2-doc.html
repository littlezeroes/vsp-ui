2. Entry Screen — Smart Input

Vấn đề v1: Quá nhiều element cạnh tranh: 3-tab filter (Tất cả/Ví/NH) + "Gần đây" (auto-recent, noise) + 2 nút nhập tay → user overwhelmed, không biết focus đâu.

Research

ZaloPayTab "Ví ZaloPay" | "Ngân hàng"Segment

Quyết định: Smart Input (auto-detect)

Gộp Cake-style + MoMo insight thành 1 input field thông minh — tự detect SĐT / STK / tên:

User nhậpPatternAuto-detectHành vi
098...Bắt đầu 0, max 10 số→ Ví VSPFilter saved P2P + lookup ví VSP khi đủ 10 số
1903...Không bắt đầu 0, hoặc >10 số→ Ngân hàngAuto-detect bank từ BIN prefix + NAPAS lookup tên
NamChữ cái→ Tìm tênSearch across tất cả saved (P2P + Bank)
Tại sao bỏ Segmented Control? Smart input xử lý cả 2 case mà không cần user chọn tab trước. Giảm 1 decision point. User chỉ cần nhập — hệ thống tự hiểu.

Flow theo loại input

  • SĐT (P2P): Nhập → auto-filter saved → đủ 10 số → resolve tên → "Tiếp tục" → Amount
  • STK (Bank): Nhập → auto-detect bank (BIN) → resolve tên (NAPAS) → "Tiếp tục" → Amount
  • Tên: Nhập → filter saved list (cả P2P + Bank) → tap chọn → Amount
  • Tap saved: Tap trực tiếp từ list → Amount (skip input hoàn toàn)

Lưu người nhận — Opt-in

Không show "Gần đây" tự động. Chỉ show những người nhận mà user đã chủ động bật toggle "Lưu người nhận". Lý do:

  • User chuyển linh tinh → lưu làm gì?
  • Chủ động lưu = danh sách sạch, đúng nhu cầu
  • Saved list hiện tất cả (P2P + Bank) — không cần tab filter
  • Empty state cho user mới: "Chưa có người nhận nào" + helper chips

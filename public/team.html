QUY TRÌNH — 3 PHASES, 11 STEPS
Phase A — Understand
~45 phút
01 Đọc BRD, phân tích gaps, tìm edge cases 🔍 Nate
02 Hỏi PO xác nhận gaps & assumptions 🤖 Vi CHECKPOINT
03 Design user flows + edge cases 🔍 Nate
04 Review flow — chê, debate, đề xuất alternative 👹 Đức
05 Show PO kết quả debate, chốt flow 🤖 Vi CHECKPOINT
Phase B — Plan
~30 phút
06 Tạo screen breakdown — components, states, transitions 🎨 Ivy
07 Check state coverage — empty, loading, error cho mọi screen 📋 Khoa
08 Show PO screens + states, approve 🤖 Vi CHECKPOINT
Phase C — Build
~2.5 giờ
09 Gen code — parallel agents, mỗi agent 3-4 screens 🎨 Ivy
10 Full QA — token check, state coverage, design rules 📋 Khoa
11 Final report — states browser + summary 🤖 Vi CHECKPOINT
THỜI GIAN THỰC HIỆN MỖI TASK
Task Agent Thời gian Ghi chú
Phân tích BRD 🔍 Nate 5-10 phút Đọc file, extract requirements, tìm gaps
PO trả lời gaps 🤖 Vi ↔ PO tùy PO Pipeline pause tại checkpoint
Design user flows 🔍 Nate 10-15 phút Flow diagrams + edge case mapping
Review & debate 👹 Đức 5-10 phút Adversarial review, đề xuất alternatives
Screen breakdown 🎨 Ivy 10-15 phút Components, props, states per screen
State coverage QA 📋 Khoa 5 phút Verify empty/loading/error/edge states
Code gen (12 screens) 🎨 Ivy ×3 30-45 phút 3 parallel agents, ~4 screens mỗi agent
Token QA + fix 📋 Khoa 10 phút Auto-fix loop: check → fix → recheck
States browser 🤖 Vi 5 phút iPhone preview + flow charts per epic
Tổng (không tính PO wait) ~3-4 giờ Từ BRD đến interactive prototype

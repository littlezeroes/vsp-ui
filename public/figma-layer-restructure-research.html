1 Bài toán

Khi design nhanh hoặc copy-paste từ nguồn khác, layer tree trong Figma thường rơi vào trạng thái "rối" — tên generic, nesting thừa, không có semantic grouping. Bài toán: restructure layers → clean hierarchy + semantic naming, giữ nguyên pixel-perfect UI.

2 Figma Plugin API — Khả năng & Giới hạn

API chính cho restructure

OperationAPIGhi chú
Rename node.name = "..." Hoạt động trên mọi node type. Trivial.
Reparent parent.appendChild(child)
parent.insertChild(i, child)		 Move node sang parent mới. Cần tính lại x/y.
Ungroup figma.ungroup(node) = Cmd+Shift+G. Không dùng được trên Instance/Component.
Group figma.group(nodes, parent) Tạo group mới, giữ absolute position.
Create Frame figma.createFrame() Tạo frame mới cho semantic section.
Delete node.remove() Cẩn thận: xóa container = xóa luôn children.

Giữ visual position khi reparent

// Lưu vị trí tuyệt đối trước khi move
const absX = node.absoluteTransform[0][2];
const absY = node.absoluteTransform[1][2];

// Reparent
newParent.appendChild(node);

// Tính lại relative position
const parentAbsX = newParent.absoluteTransform[0][2];
const parentAbsY = newParent.absoluteTransform[1][2];
node.x = absX - parentAbsX;
node.y = absY - parentAbsY;

Ungroup ComponentNode Figma throw error — component phải giữ nguyên
Undo granular Plugin API không có undo — chỉ dựa vào Figma version history
figma.notify() Không hoạt động trong use_figma runtime

4 MCP Approaches

A. Official Figma MCP (use_figma)

Đây là tool mình đang có sẵn trong Claude Code. Chạy Plugin API JavaScript trực tiếp trong Figma. Có thể rename, reparent, ungroup, regroup — đủ cho restructure.
ƯuNhược
Đã tích hợp sẵn, không cần cài thêm Mỗi call = 1 network round-trip → chậm với nhiều layer
Claude phân tích + quyết định semantic naming Cần paid Figma plan cho write access
Incremental workflow: đọc → plan → execute → verify Beta — API có thể thay đổi
Screenshot verify sau mỗi bước Không có transactional rollback

B. claude-talk-to-figma-mcp (Community)

Approach phù hợp nhất cho restructure. Có sẵn các tool chuyên biệt: rename_node, group_nodes, ungroup_nodes, insert_child, delete_node, get_document_info.
ƯuNhược
Tool granular: rename, move, group, ungroup riêng biệt Cần cài thêm: Figma Plugin + MCP server
Hoạt động trên mọi Figma account (kể cả free) WebSocket bridge — cần plugin chạy trong Figma
get_document_info → full tree cho AI analyze Maintenance: community project, có thể outdated
Không cần paid plan

C. figma-console-mcp (Community)

Nuclear option — chạy arbitrary JavaScript. Tool figma_execute cho phép gửi script Plugin API phức tạp, chạy atomic bên trong Figma. Mạnh nhất nhưng cần biết Plugin API.
ƯuNhược
Chạy bất kỳ Plugin API code nào Cần cài Desktop Bridge plugin
Script atomic — lỗi thì không thay đổi gì 90+ tools — phức tạp
Batch operations trong 1 call Community project

5 Rủi ro & Mitigation

Rủi roKhi nào xảy raMức độMitigation
Auto-layout bể Reparent node giữa các auto-layout frame Cao Check layoutMode trước. Set layoutPositioning: "ABSOLUTE" nếu cần.
Position lệch Move child ra khỏi frame có offset Trung bình Dùng absoluteTransform để tính lại x/y sau khi reparent.
Fill/stroke mất Xóa wrapper frame có background/border Trung bình Check fills, strokes, effects trước khi ungroup. Chỉ ungroup frame "trống".
Instance hỏng Sửa internal structure của component instance Cao KHÔNG bao giờ sửa bên trong instance. Skip hoặc detach trước.
Mất children node.remove() xóa container Cao Luôn move children ra trước, rồi mới remove container rỗng.
Không undo được Script chạy xong, kết quả sai Trung bình Dựa vào Figma version history. Hoặc duplicate screen trước khi restructure.
Rule vàng: Luôn DUPLICATE screen trước khi restructure. Đặt tên "[Backup] Hop dong — before restructure". Nếu mọi thứ bể, vẫn còn bản gốc.

6 Best Practices — Layer Organization

Naming Convention

PatternVí dụKhi dùng
Role-basedHeader, Content, FooterPage sections
Section prefix[Section] Contract listingSemantic groups
Screen prefix[Screen] Hop dongTop-level screens
Element typeContract card, Tab, DividerUI components
State suffixContract card – Pending, – CompleteVariants/states
Slash hierarchyButton/Primary/DefaultDesign system components
Anti-patterns: Frame 123, Rectangle 45, Group 7, Copy, Copy 2 — Figma defaults, vô nghĩa, không bao giờ nên giữ.

Hierarchy Depth

  • Target: 3–5 levels max
  • Mỗi level nesting = nhân layout calculations (đặc biệt với nested components)
  • Rule: Nếu frame chứa đúng 1 child + không có fill/stroke/effect → wrapper thừa, xóa đi
  • Deep nesting → performance drop đáng kể trong Figma

Tương lai: Custom Skill

Nếu cần restructure nhiều screen, nên build 1 custom Claude Code skill (.claude/commands/restructure-figma.md) tự động hóa toàn bộ workflow. Có thể tái sử dụng cho mọi screen trong project.
